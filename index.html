Making a Webpage Go Live with Git!

The Tools

In order to perform this incredible task, first you must have the correct tools. These tools include:

  • A Github account
  • Github for Desktop
  • Atom

With these tools, you can change the world!

The Process

  1. Firstly, sign into your Github account on your browser. This is imperative to the rest of the process. Ensure that you have verified your email for maximum efficiency.
  2. Secondly, create a brand spankin' new repository for your website on Github. Optionally, include a README file.
  3. Next, you'll need to sign into Github for desktop with the same account. This will allow you to link the online Github server with your local desktop.
  4. Afer that, you must clone that repository from Step 2. Do not add or create in this situation, which would surely lead to only disaster. Save it to a convenient and logical place for your purposes.
  5. Now, switch to Atom, and create yourself an "index.html" file. Or perhaps you already have one. In either case, it should be saved into the cloned repository, located in the "git" folder, wherever you happened to save that.
  6. Once this has been done, you'll want to switch back to Github for Desktop. If everything has gone according to plan, then under the "Changes" tab, you should have an astonishing "1" icon. This will show you your new file. After a quick double check to make sure everything is in order, give your change a name, and press the "commit to master" button.
  7. Next, smash that "sync" button on the top right and wait for it to do its thing.
  8. After the sync has completed, flip back to the Github in your browser and give it a cheeky refresh. Your local file should now appear on their server. If you enter the name of your repository into the URL bar of your browser, it should take you to your freshly coded page.
  9. Congratulate yourself. Your webpage is now live. You're on your way to a fruitful and fulfilling life of web design!

And there you have it, that's how you make a webpage go live with Git. Pretty simple, huh?
